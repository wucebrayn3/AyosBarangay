
Below is the current backend API list for testing. Use base URL:

http://192.168.0.23:8000 or http://127.0.0.1:8000

For protected requests, add:

Authorization: Bearer YOUR_ACCESS_TOKEN
Content-Type: application/json
Recommended Test Order

POST /api/auth/register/
POST /api/auth/login/
Copy the access token.
Use Authorization: Bearer <access> for protected APIs.
Use an admin, staff, or purok_leader account for management endpoints like puroks, issues, concerns, and assignments.
Auth
POST /api/auth/register/

For creating a resident account.

{
  "username": "resident1",
  "password": "StrongPass123!",
  "password_confirm": "StrongPass123!",
  "first_name": "Resident",
  "last_name": "One",
  "email": "resident1@example.com",
  "purok": "Purok 1",
  "phone_number": "09170000001"
}
POST /api/auth/login/

For logging in and getting JWT tokens.

{
  "username": "resident1",
  "password": "StrongPass123!"
}
POST /api/auth/refresh/

For getting a new access token.

{
  "refresh": "YOUR_REFRESH_TOKEN"
}
GET /api/auth/me/

For checking the logged-in user.

curl http://127.0.0.1:8000/api/auth/me/ -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
POST /api/auth/logout/

For logging out by blacklisting the refresh token.

{
  "refresh": "YOUR_REFRESH_TOKEN"
}
Users
GET /api/users/

For listing registered users. Public in current config.

POST /api/users/

For creating users manually. Admin only.

{
  "username": "staff1",
  "first_name": "Staff",
  "last_name": "User",
  "email": "staff1@example.com",
  "role": "staff",
  "purok": "Purok 1",
  "phone_number": "09171111111"
}
GET /api/users/{id}/

For viewing one user.

PATCH /api/users/{id}/

For updating user info/role. Admin only.

{
  "role": "purok_leader"
}
DELETE /api/users/{id}/

For deleting a user. Admin only.

Infrastructure Issues
GET /api/infrastructure-issues/

For listing infrastructure reports.

Useful filters:
/api/infrastructure-issues/?status=pending
/api/infrastructure-issues/?category=pothole
/api/infrastructure-issues/?purok=1
/api/infrastructure-issues/?search=streetlight
/api/infrastructure-issues/?ordering=-created_at

POST /api/infrastructure-issues/

For reporting infrastructure problems. Requires staff, admin, or purok_leader.

{
  "reporter": 1,
  "title": "Pothole near chapel",
  "description": "Deep pothole causing traffic risk.",
  "category": "pothole",
  "status": "pending",
  "purok": 1,
  "latitude": "14.599500",
  "longitude": "120.984200",
  "address_text": "Main road near chapel",
  "is_verified": false,
  "is_public": true,
  "is_emergency": false,
  "deadline": "2026-05-20"
}
GET /api/infrastructure-issues/{id}/

For viewing one infrastructure issue.

PATCH /api/infrastructure-issues/{id}/

For updating status, verification, visibility, etc.

{
  "status": "in_progress",
  "is_verified": true
}
DELETE /api/infrastructure-issues/{id}/

For deleting an infrastructure issue.

Got you. For Postman, use this setup:

Base URL:
http://192.168.0.23:8000

Create a Postman variable:
base_url = http://192.168.0.23:8000

After login, create variables:
access_token = <access token>
refresh_token = <refresh token>

For protected endpoints, set Headers:

Authorization: Bearer {{access_token}}
Content-Type: application/json
For image upload endpoints, do not manually set Content-Type; use Body > form-data.
